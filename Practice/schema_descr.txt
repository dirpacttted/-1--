Структура БД:

1. Таблица Configurations:
Хранит информацию о конфигурациях.

Поля:
- id – INT, первичный ключ, автоинкремент.
- name – VARCHAR(255), имя конфигурации.
- synonym – VARCHAR(255), синоним конфигурации.
- default_download_path – VARCHAR(500), путь к каталогу загрузки по умолчанию.

Связь:
Одна конфигурация может иметь несколько версий.


2. Таблица ConfigurationVersions:
Хранит различные версии конфигураций.

Поля:
- id – INT, первичный ключ.
- configuration_id – INT, внешний ключ.
- version – VARCHAR(100), номер версии.

Связь:
Принадлежит одной конфигурации и может содержать множество объектов метаданных.


3. Таблица MetadataObjects:
Хранит объекты метаданных.

Поля:
- id – INT, первичный ключ.
- version_id – INT, внешний ключ.
- object_type – VARCHAR(100), тип объекта.
- name – VARCHAR(255), имя объекта.
- synonym – VARCHAR(255), синоним.
- object_uuid – CHAR(36), уникальный идентификатор объекта.

Связь:
Каждый объект метаданных принадлежит одной версии конфигурации.


4. Таблица DataSeparationSettings:
Хранит параметры разделения данных.

Поля:
- id – INT, первичный ключ.
- metadata_object_id – INT, внешний ключ.
- auxiliary_data_area – VARCHAR(255), значение реквизита "ОбластьДанныхВспомогательныеДанные".
- main_data_area – VARCHAR(255), значение реквизита "ОбластьДанныхОсновныеДанные".

Связь:
Каждая запись относится к одному объекту метаданных.
