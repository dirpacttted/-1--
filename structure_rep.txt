Отчёт проверки структуры БД

Результат проверки:
Структура базы данных соответствует требованиям 1-го этапа.

Проверенные таблицы:
1. Configurations:
Хранение информации о конфигурациях.

Проверенные поля:
- id — первичный ключ.
- name — имя конфигурации.
- synonym — синоним конфигурации.
- default_download_path — путь загрузки по умолчанию.


2. ConfigurationVersions:
Хранение версий конфигураций.

Проверенные поля:
- id — первичный ключ.
- configuration_id — внешний ключ на таблицу Configurations.
- version — версия конфигурации.


3. MetadataObjects:
Хранение объектов метаданных.

Проверенные поля:
- id — первичный ключ.
- version_id — внешний ключ на таблицу ConfigurationVersions.
- object_type — тип объекта.
- name — имя объекта.
- synonym — синоним объекта.
- object_uuid — уникальный идентификатор объекта.


4. DataSeparationSettings:
Хранение параметров разделения данных.

Проверенные поля:
- id — первичный ключ.
- metadata_object_id — внешний ключ на таблицу MetadataObjects.
- auxiliary_data_area — область данных вспомогательных данных.
- main_data_area — область данных основных данных.


Связи между таблицами:

Configurations → ConfigurationVersions (1:N)

ConfigurationVersions → MetadataObjects (1:N)

MetadataObjects → DataSeparationSettings (1:1)

Итог:
Все необходимые таблицы созданы.
Определены первичные и внешние ключи.
Связи между таблицами соответствуют структуре базы данных.
